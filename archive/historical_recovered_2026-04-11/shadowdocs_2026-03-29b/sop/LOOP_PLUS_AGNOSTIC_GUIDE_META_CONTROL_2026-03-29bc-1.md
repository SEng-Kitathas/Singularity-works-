# Loop+ Agnostic Guide — Meta Control Layer Updated
_Version: 2026-03-29bc_

## Purpose
This document defines the **meta control layer** for Loop+ in a project-agnostic way.

Loop+ is still the outer research loop:
1. primary-source pass
2. discovery expansion
3. fast-signal / implementation-clue pass
4. isomorph hunting
5. native translation
6. control-layer update
7. experiment pressure
8. recursive replay

But all of that is now governed by a stricter extraction/control spine:

> **OIE + ELT + Systematic Extraction + DEFNLP-style Post-Processing**

This meta layer applies to:
- uploaded corpora
- web research
- benchmark packets
- design threads
- standards / docs / news bundles
- any external evidence entering the project

---

## 1. Meta Control Layer

## 1.1 OIE layer
Use an Open Information Extraction mindset to maximize fact capture without assuming a narrow predefined schema.

Capture:
- entities
- relations
- claims
- dependencies
- contradictions
- process rules
- benchmark implications
- doctrine changes

Recommended normalization:
- subject
- predicate
- object
- confidence / evidence strength
- source pointer

Goal:
- prevent hidden facts from remaining trapped in prose
- make external evidence queryable and recomposable

---

## 1.2 ELT layer
Use ELT logic for evidence handling:

**Extract → Load → Transform**

Rule:
- preserve raw material first
- manifest it
- hash it if practical
- only then transform, summarize, or reinterpret

This prevents losing signal because of premature compression.

Required outputs when justified:
- raw manifest
- corpus map
- file inventory
- checksums / IDs where practical
- structured transformed outputs downstream

Goal:
- future-proof the corpus
- allow re-interpretation later
- prevent irreversible filtering mistakes

---

## 1.3 Systematic Extraction layer
Treat the corpus like a structured review process, especially when the evidence is broad or high-stakes.

For each meaningful evidence pass:
- predefine what variables / seams you are extracting
- use a standardized extraction matrix
- distinguish:
  - new signal
  - reinforcement
  - contradiction
  - correction
  - unresolved seam
- make disagreement handling explicit if there are competing interpretations

Recommended matrix fields:
- source
- seam
- claim
- evidence strength
- doctrine effect
- status
- next action

Goal:
- reduce omission
- reduce inconsistency
- reduce hindsight distortion

---

## 1.4 DEFNLP-style post-processing layer
After primary extraction, run a rescue / clean-up pass.

This should look for:
- orphan signal
- missed acronyms / aliases
- repeated claims under different wording
- hidden dependencies
- baseline mistakes
- category errors
- facts stranded in appendices / side notes / implementation notes
- contradictions between local and global summaries

Goal:
- catch what the first pass missed
- prevent summary loss
- tighten the doctrine delta before promotion

---

## 2. Loop+ Under the Meta Control Layer

## Stage A — Research
Research broadly across:
- primary sources
- discovery infra
- practical/fast-signal sources
- internal corpus
- cross-domain isomorphs

## Stage B — Extract
Run:
- OIE-style fact extraction
- ELT-style preservation
- systematic extraction matrix

## Stage C — Rescue
Run:
- DEFNLP-style post-processing
- duplicate/alias merge
- contradiction recovery
- baseline correction recovery

## Stage D — Translate
Convert outside signal into:
- project-neutral operational terms
or
- project-native ontology if inside an existing project

## Stage E — Pressure
Use the translated findings to:
- update control docs
- change queue order
- define experiments
- demote stale assumptions
- promote load-bearing findings

## Stage F — Recurse
Replay earlier seams if the new evidence changes interpretation.

---

## 3. Default Artifact Set

Every serious Loop+ pass should leave behind:

1. **manifest**
2. **OIE/triples table** (when useful)
3. **standardized extraction matrix**
4. **post-process synthesis**
5. **main synthesis / doctrine delta**
6. **package archive** (when the corpus is large enough)

This is the default whether the input came from:
- uploads
- websites
- preprint repositories
- papers
- news / implementation signal
- mixed bundles

---

## 4. Minimal Operating Questions

For any Loop+ corpus, answer these:

1. What is genuinely new signal?
2. What only reinforces what we already knew?
3. What contradicts or weakens current doctrine?
4. What was a baseline or interpretation mistake?
5. What is still unresolved?
6. What should be promoted into the control layer?
7. What should become experiment pressure?
8. What older conclusion must be replayed recursively?

---

## 5. Anti-Failure Rules

- Do not summarize before preserving raw structure.
- Do not treat one source family as sufficient.
- Do not let fast-signal outlets masquerade as primary mechanism proof.
- Do not let extraction stop at the first clean summary.
- Do not leave doctrine-changing facts trapped in prose.
- Do not fail to distinguish evidence from interpretation.
- Do not fail to record what remains unresolved.

---

## 6. Short Form

Loop+ now runs under this meta control law:

> **Research broadly → preserve raw evidence → extract facts openly → standardize claims → rescue orphan signal → translate locally → pressure experiments → replay recursively**

Or even shorter:

> **OIE + ELT + Systematic Extraction + DEFNLP-style rescue**  
> wrapped around the normal research loop.
