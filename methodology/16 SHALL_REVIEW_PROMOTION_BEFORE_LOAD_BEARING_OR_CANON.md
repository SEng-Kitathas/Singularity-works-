# Promotion Review
_Version: 2026-03-31i_

## Purpose
Use this instrument before elevating a claim, pattern, workflow, artifact, or doctrine fragment into a trusted load-bearing role.

## Operator / lab meaning
This file is the promotion gate.
It exists to prevent premature canonization, status inflation, and accidental conversion of a useful seed into false doctrine.

## Model / control obligations
The model SHALL:
- treat promotion as earned status, not narrative convenience
- review contradiction survival, embodiment, verification, cost, and generalization range
- distinguish "works here" from "works broadly"
- state whether the candidate reduces global cost or merely relocates it

The model SHALL NOT:
- promote because an idea is elegant, favored, or repeatedly discussed
- confuse local success with reusable doctrine
- skip explicit reasoning about complexity debt
