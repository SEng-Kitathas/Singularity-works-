# Zero-Loss Extraction Instrument
_Version: 2026-03-31i_

## Purpose
Use this instrument when a corpus, archive, transcript, repo, article bundle, or evidence mass is large enough that normal summarization would likely erase signal.

## Operator / lab meaning
This file is the anti-compression membrane.
Its job is to preserve seams, contradictions, repeated motifs, corrections, and latent structure before synthesis collapses them into a cleaner but less truthful story.

## Model / control obligations
The model SHALL:
- inventory the corpus before drawing conclusions
- separate raw evidence contact from derived interpretation
- preserve contradiction, repetition, drift, and uncertainty
- record what was actually read or inspected
- distinguish new signal from reinforcement of known signal
- emit replay candidates where another pass may reveal missed structure

The model SHALL NOT:
- summarize first and inspect later
- merge distinct seams because they feel related
- hide unread sections behind fluent synthesis
- silently discard contradiction because a cleaner narrative exists

## When to use
Use this file when:
- a corpus is broad, dense, or historically layered
- multiple versions or branches exist
- a repo or archive must be read for drift, doctrine, or architecture
- the cost of losing weak signal is high

## Known failure modes this file suppresses
- summary-first distortion
- missed weak signal
- version drift blindness
- conflation of reinforcement with novelty
- fake completion from partial reads

## Update rule
This file should be appended or forked when a later extraction pass materially changes what the corpus is understood to contain.

## Normative note
This instrument SHALL be used when broad corpora or high-signal evidence bundles risk compression loss. A clean summary is not a substitute for extraction fidelity.
